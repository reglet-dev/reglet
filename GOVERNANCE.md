# Governance Policy

## Overview

This project is led by the maintainers and driven by the community. We strive to create a welcoming, inclusive, and productive environment for everyone.

## Roles

### Contributors
Everyone who contributes code, documentation, or other work to the project is a contributor. Reviewing pull requests and active involvement in discussions are also highly valued contributions.

### Maintainers
Maintainers are contributors who have significant responsibilities for the project's direction and repository management. Maintainers:
- Can merge Pull Requests.
- Can close Issues and Pull Requests.
- Drive the roadmap and technical direction.

## Decision Making

Decisions are made based on consensus among maintainers. Major changes should be discussed in GitHub Issues or Discussions before implementation.

## Code of Conduct

All participants must adhere to our [Code of Conduct](CODE_OF_CONDUCT.md). Instances of abusive, harassing, or otherwise unacceptable behavior may be reported by contacting the project team.
